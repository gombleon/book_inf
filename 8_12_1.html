8.12.1. Рекурсия

Объект называется рекурсивным, если он содержит сам себя или определен с помощью самого себя. Приведем примеры рекурсивных определений натуральных чисел, древовидных структур и некоторых функций из [41], с. 116 — 119:

  1. Натуральные числа:
    1. 1 есть натуральное число;
    2. целое число, следующее за натуральным, есть натуральное число.
  2. Древовидные структуры.
  3. Функция факториал n! для неотрицательных целых чисел:
    1. 0!=1;
    2. если n>0, то n! = n⁢ (n-1)!

Рекурсия позволяет определить бесконечное множество объектов посредством конечного высказывания. С помощью рекурсивных алгоритмов можно описать бесконечные вычисления. Особенно полезны рекурсии в тех случаях, когда вычисляемая функция или обрабатываемая структура определены посредством рекурсии (рекурсивны по своей природе).

Рекурсивное определение программы реализуется через процедуры, в которых используется вызов именованных операторов. Рекурсивную программу P можно представить как композицию Q базовых операторов Si языка программирования, не содержащих P, и самой программы P: P≡Q⁡SiP.

Если процедура P содержит явное обращение к самой себе, то она называется прямо рекурсивной; если же P содержит обращение к процедуре T, которая содержит, прямо или косвенно, обращение к P, то P называется косвенно рекурсивной.

В процедуре задается множество локальных констант, типов, переменных и процедур. Всякий рекурсивный вызов процедуры приводит к созданию в своей области видимости нового множества локальных констант, типов, переменных и процедур, имеющих те же самые имена, но разные значения. Поэтому в целях экономии вычислительных ресурсов необходимо стремиться к использованию рекурсивных процедур с наименьшей глубиной.

Рекурсивные процедуры, как и циклы, могут привести к бесконечным вычислениям. Следовательно, рекурсивное выполнение процедуры необходимо подчинить некоторому условию, которое обязательно через некоторое количество операций перестает выполняться и прерывает рекурсивный вызов процедуры.

При анализе рекурсивной программы возникает, как обычно, два вопроса:

  1. почему программа заканчивает работу?
  2. почему она работает правильно, если заканчивает работу?

Для (b) достаточно проверить, что (содержащая рекурсивный вызов) программа работает правильно, предположив, что вызываемая ею одноименная программа работает правильно. В самом деле, в этом случае в цепочке рекурсивно вызываемых программ все программы работают правильно (убеждаемся в этом, идя от конца цепочки к началу).

Чтобы доказать (a), обычно проверяют, что с каждым рекурсивным вызовом значение какого-то параметра уменьшается, и это не может продолжаться бесконечно.

Перейдем рассмотрению примеров рекурсивных алгоритмов.

Пример 8.12.1.1. Рассмотрим рекурсивную программу вычисления факториала натурального числа n:

MODULE fact1;

IMPORT Log := StdLog, In := i21sysIn, Math;

PROCEDURE factorial(n: INTEGER; VAR fact: INTEGER);

VAR

BEGIN

IF n=1 THEN

fact:=1;

ELSE

factorial(n-1, fact);

fact:=n*fact;

END;

END factorial;

PROCEDURE do*;

VAR n, fact: INTEGER;

BEGIN

In.Open;

In.Int(n);

ASSERT(In.Done & (n > 0));

factorial(n, fact);

Log.Int(fact);

Log.Ln

END do;

END fact1.

Обратите внимание на формальные параметры в процедуре factorial⁡ ( n: INTEGER ; VAR fact: INTEGER ) , в которой передача параметра n происходит по значению, а параметра fact — по ссылке.

Пример 8.12.1.2. Рассмотрим рекурсивную программу возведения целого числа a в целую неотрицательную степень n:

MODULE степень;

IMPORT Log := StdLog, In := i21sysIn;

PROCEDURE step(a, n: INTEGER): INTEGER;

VAR s: INTEGER;

BEGIN

IF (n=0) THEN

s:=1

ELSIF ODD(n) THEN

s:=step(a, n-1)*a

ELSE

s:=step(a*a, n DIV 2)

END;

RETURN s;

END step;

PROCEDURE do*;

VAR stepen, a, n: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.Int(n);

ASSERT(In.Done & (n ≥ 0));

In.Int(a);

ASSERT(In.Done);

stepen:=step(a, n);

Log.Int(a);

Log.String('^');

Log.Int(n);

Log.String(' = ');

Log.Int(stepen);

Log.Ln

END do;

END степень.

Нетрудно убедиться, что в приведенной выше рекурсивной программе имеет место оценка глубины рекурсии C⁢log⁢n.

Пример 8.12.1.3.Рекурсивная программа записи цифр натурального числа:

MODULE цифры;

IMPORT Log := StdLog, In := i21sysIn;

PROCEDURE do*;

VAR n: INTEGER;

PROCEDURE print(n: INTEGER);

VAR

BEGIN

IF n < 10 THEN

Log.Int(n)

ELSE

print(n DIV 10);

Log.Int(n MOD 10)

END;

END print;

BEGIN

In.Open;

In.Int(n);

ASSERT(In.Done & (n > 0));

Log.Ln;

Log.String('Число ');

Log.Int(n);

Log.Ln;

Log.String('Цифры: ');

print(n)

END do;

END цифры.

Пример 8.12.1.4. Игра "Ханойские башни" состоит в следующем. Есть три стержня. На первый из них надета пирамидка из N колец (большие кольца снизу, меньшие сверху). Требуется переместить кольца на другой стержень. Разрешается перекладывать кольца со стержня на стержень, но класть большее кольцо поверх меньшего нельзя. Программа, указывающая требуемые действия, такая:

MODULE hanoi;

IMPORT Log := StdLog, In := i21sysIn;

PROCEDURE do*;

CONST x=1; y=2; z=3;

VAR n: INTEGER;

PROCEDURE move(n, x, y, z: INTEGER);

BEGIN

IF (n > 0) THEN

move(n-1, x, z, y);

Log.String('ход ');

Log.Int(x);

Log.String(' → ');

Log.Int(y);

Log.Ln;

move(n-1, z, y, x)

END;

END move;

BEGIN

In.Open;

In.Int(n);

ASSERT(In.Done & (n > 0));

Log.Ln;

Log.String('Всего колец ');

Log.Int(n);

Log.Ln;

move(n, x, y, z)

END do;

END hanoi.

  4. Даны два целых числа A и В. Составить рекурсивную программу вывода всех чисел от A до B включительно, в порядке возрастания, если A < B, или в порядке убывания в противном случае.
  5. В теории вычислимости важную роль играет функция Аккермана A⁡(m,n), определенная следующим образом: A⁡ (m,n) = { n+1,m=0; A⁡ (m-1,1) ,n=0,m>0; A⁡ (m-1, A⁡ (m,n-1) ) ,n>0,m>0. По заданным целым неотрицательным m и n выведите A⁡(m,n).
  6. Дано натуральное число N. Разработать рекурсивную программу, выводящую слово YES, если число N является точной степенью двойки, и слово NO — в противном случае.
  7. Дано натуральное число N. Вычислите сумму его цифр. При решении этой задачи нельзя использовать строки, списки, массивы и циклы).
  8. Дано натуральное число N. Выведите все его цифры по одной, в обратном порядке, разделяя их пробелами или новыми строками. При решении этой задачи нельзя использовать строки, списки, массивы и циклы. Разрешена только рекурсия и целочисленная арифметика.
  9. Дано натуральное число N. Выведите все его цифры по одной, в обычном порядке, разделяя их пробелами или новыми строками. При решении этой задачи нельзя использовать строки, списки, массивы и циклы. Разрешена только рекурсия и целочисленная арифметика.
  10. Дано натуральное число n, большее единицы. Проверьте, является ли оно простым. Программа должна вывести слово YES, если число простое, и NO, если число составное. Алгоритм должен иметь сложность O⁡(log⁡n).
    Указание. Понятно, что задача сама по себе нерекурсивна, т.к. проверка числа n на простоту никак не сводится к проверке на простоту меньших чисел. Поэтому нужно сделать еще один параметр рекурсии: делитель числа, и именно по этому параметру и задать рекурсию.
  11. Дано натуральное число n, большее единицы. Выведите все простые множители этого числа в порядке неубывания с учетом кратности. Алгоритм должен иметь сложность O⁡(log⁡n).
  12. Дано слово, состоящее только из строчных латинских букв. Проверьте, является ли это слово палиндромом. Выведите YES или NO. При решении этой задачи нельзя пользоваться циклами.
  13. Дана последовательность натуральных чисел, завершающаяся числом 0. Выведите все нечетные числа из этой последовательности, сохраняя их порядок.
    В этой задаче нельзя использовать глобальные переменные и передавать какие-либо параметры в рекурсивную функцию. Функция не возвращает значение, а сразу же выводит результат на экран. Основная программа должна состоять только из вызова этой функции.
  14. Дана последовательность натуральных чисел (одно число в строке), завершающаяся числом 0. Выведите первое, третье, пятое и т.д. из введенных чисел. Завершающий ноль выводить не надо.
    В этой задаче нельзя использовать глобальные переменные и передавать какие-либо параметры в рекурсивную функцию. Функция не возвращает значение, а сразу же выводит результат на экран. Основная программа должна состоять только из вызова этой функции.
  15. Дана последовательность натуральных чисел (одно число в строке), завершающаяся числом 0. Определите наибольшее значение числа в этой последовательности.
    В этой задаче нельзя использовать глобальные переменные и передавать какие-либо параметры в рекурсивную функцию. Функция возвращает единственное значение: максимум считанной последовательности. Гарантируется, что последовательность содержит хотя бы одно число (кроме нуля).
  16. Дана последовательность натуральных чисел (одно число в строке), завершающаяся числом 0. Определите среднее значение элементов этой последовательности (без учета последнего нуля).
    В этой задаче нельзя использовать глобальные переменные. Функция не получает числа в виде параметра. Результат записывается в две переменные, которые передаются в функцию по ссылке.
    Гарантируется, что последовательность содержит хотя бы одно число (кроме нуля).
  17. Дана последовательность натуральных чисел (одно число в строке), завершающаяся числом 0. Определите значение второго по величине элемента в этой последовательности, то есть элемента, который будет наибольшим, если из последовательности удалить наибольший элемент.
    В этой задаче нельзя использовать глобальные переменные. Функция не получает числа в виде параметра и вообще не получает никаких параметров. Результат записывается в переменные, которые передаются в функцию по ссылке. Других параметров, кроме как используемых для возврата значения, функция не получает.
    Гарантируется, что последовательность содержит хотя бы два числа (кроме нуля).
  18. Дана последовательность натуральных чисел (одно число в строке), завершающаяся числом 0. Определите, какое количество элементов этой последовательности, равны ее наибольшему элементу.
    В этой задаче нельзя использовать глобальные переменные. Функция не получает числа в виде параметра и вообще не получает никаких параметров. В программе результат записывается в переменные, которые передаются в функцию по ссылке. Других параметров, кроме как используемых для возврата значения, функция не получает.
    Гарантируется, что последовательность содержит хотя бы одно число (кроме нуля).
  19. Дана последовательность натуральных чисел (одно число в строке), завершающаяся двумя числами 0 подряд. Определите, сколько раз в этой последовательности встречается число 1. Числа, идущие после двух нулей, необходимо игнорировать.
    В этой задаче нельзя использовать глобальные переменные и параметры, передаваемые в функцию. Функция не получает числа в виде параметров.

    По данному натуральному n выведите первые n членов этой последовательности. Попробуйте обойтись только одним циклом for.
  21. Даны натуральные числа k и s. Определите, сколько существует k-значных натуральных чисел, сумма цифр которых равна d. Запись натурального числа не может начинаться с цифры 0. В этой задаче можно использовать цикл для перебора всех цифр, стоящих на какой-либо позиции.
  22. Даны числа a и b. Определите, сколько существует последовательностей из a нулей и b единиц, в которых никакие два нуля не стоят рядом.
  23. Дано число n, десятичная запись которого не содержит нулей. Получите число, записанное теми же цифрами, но в противоположном порядке.
