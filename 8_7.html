8.7. Циклы

Оператор цикла FOR описывает выполнение операторной последовательности, повторяющееся для последовательности значений, присваиваемых целой переменной, называемой управляющей переменной цикла (или индексом).

Синтаксис оператора цикла FOR:

FOR индекс:= начальное_значение TO конечное_значение [BY шаг] DO

последовательность_операторов

END;

Оператор цикла WHILE описывает выполнение операторной последовательности, повторяющееся, пока некое логическое выражение (его охрана) дает TRUE. Охрана проверяется заново перед каждым вычислением операторной последовательности..

Синтаксис оператора цикла WHILE:

WHILE охрана DO

последовательность_операторов

END;

Оператор цикла REPEAT описывает выполнение операторной последователь­ности, повторяющееся до тех пор, пока некое логическое выражение не даст TRUE. Последовательность выполняется хотя бы один раз.

Синтаксис оператора цикла REPEAT:

REPEAT

последовательность_операторов

UNTIL логическое_выражение;

Оператор LOOP описывает повторяющееся выполнение операторной после­довательности. Оно прекращается выполнением оператора выхода EXIT внутри этой последовательности.

Синтаксис оператора цикла LOOP:

LOOP

последовательность_операторов

END;

Пример 8.7.1. Дано целое число а и натуральное (целое положительное) число n. Составить программу вычисления степени an.

Самый простой вариант программы:

MODULE степень;

IMPORT Log := StdLog, In := i21sysIn, Math;

PROCEDURE do*;

VAR stepen, a, n, i: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.Int(n);

ASSERT(In.Done & (n>0));

In.Int(a);

ASSERT(In.Done);

stepen:=1;

i:=0;

WHILE (i < n) DO

stepen:=stepen*a;

INC(i)

END;

Log.Int(a);

Log.String('^');

Log.Int(n);

Log.String(' = ');

Log.Int(stepen);

Log.Ln

END do;

END степень.

Для реализации программы требуется n операций присваивания и n+1 операция сравнения.

Известно, что компьютер операцию сравнения двух чисел выполняет быстрее, если одно из чисел совпадает с нулем. Поэтому более оптимальный вариант программы такой:

MODULE степень1;

IMPORT Log := StdLog, In := i21sysIn, Math;

PROCEDURE do*;

VAR stepen, a, n, i: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.Int(n);

ASSERT(In.Done & (n>0));

In.Int(a);

ASSERT(In.Done);

stepen:=1;

i:=n;

WHILE (i#0) DO

stepen:=stepen*a;

DEC(i)

END;

Log.Int(a);

Log.String('^');

Log.Int(n);

Log.String(' = ');

Log.Int(stepen);

Log.Ln

END do;

END степень1.

Самый оптимальный вариант программы:

MODULE степень1;

IMPORT Log := StdLog, In := i21sysIn, Math;

PROCEDURE do*;

VAR stepen, a, n, i, b: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.Int(n);

ASSERT(In.Done & (n>0));

In.Int(a);

ASSERT(In.Done);

stepen:=1;

i:=n;

b:=a;

WHILE (i#0) DO

IF (i MOD 2=0) THEN

b:=b*b;

i:=i DIV 2;

ELSE

stepen:=stepen*b;

DEC(i)

END;

END;

Log.Int(a);

Log.String('^');

Log.Int(n);

Log.String(' = ');

Log.Int(stepen);

Log.Ln

END do;

Каждый второй раз (не реже) будет выполняться первый вариант оператора выбора (если k нечётно, то после вычитания единицы становится чётным), так что за два цикла величина k уменьшается по крайней мере вдвое. Тем самым для вычисления степени an потребуется C ⁡ log2 ⁡ n операций присваивания для некоторой константы C. Такого же порядка будет количество операций сравнения. Другими словами, при росте показателя n количество операций присваивания (сравнения) будет расти с логарифмической скоростью.

Наименьшее количество операций присваивания, равное log2 ⁡ n , потребуется в случае, когда показатель n есть степень числа 2.

Таким образом, для функции сложности алгоритма справедлива асимптотическая оценка f⁡(n) ∈ θ⁡( log2 ⁡ n ) .

Пример 8.7.2. Составить программу вычисления наибольшего общего делителя заданных целых неотрицательных чисел a и b, не равных нулю одновременно.

Самый простой вариант программы такой:

MODULE z13;

IMPORT Log := StdLog, In := i21sysIn, Math;

PROCEDURE do*;

VAR a, b, k: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.Int(a);

ASSERT(In.Done & (a>=0));

In.Int(b);

ASSERT(In.Done & (b>0));

IF a>b THEN

k:=a

ELSE

k:=b

END;

WHILE ~((a MOD k=0) & (b MOD k=0)) DO

DEC(k)

END;

Log.Int(k);

Log.Ln

END do;

END z13.

Функция временной сложности программы будет асимптотически ограничена сверху некоторой линейной функцией от максимума заданных чисел a и b.

В следующем варианте программы используем цикл REPEAT, что несколько упростит охрану цикла:

MODULE z13_1;

IMPORT Log := StdLog, In := i21sysIn, Math;

PROCEDURE do*;

VAR a, b, k: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.Int(a);

ASSERT(In.Done & (a>=0));

In.Int(b);

ASSERT(In.Done & (b>0));

IF a>b THEN

k:=a

ELSE

k:=b

END;

REPEAT

DEC(k);

UNTIL (a MOD k=0) & (b MOD k=0);

Log.Int(k);

Log.Ln

END do;

END z13_1.

Рассмотрим алгоритм Евклида вычисления наибольшего общего делителя НОД⁡ab натуральных чисел a и b. Без ограничения общности полагаем, что a≥b. Последовательным вычитанием из числа a число b получим представление a= b⁢d+r , где r<b. Если r=0, то НОД⁡ab = b , в противном случае НОД⁡ab = НОД⁡br , и задача сводится к нахождению НОД⁡br. Описанный процесс продолжается до тех пор, пока не получим искомое значение НОД⁡ab.

Вариант реализации алгоритма Евклида:

MODULE z13_2;

IMPORT Log := StdLog, In := i21sysIn, Math;

PROCEDURE do*;

VAR a, b, k: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.Int(a);

ASSERT(In.Done & (a>0));

In.Int(b);

ASSERT(In.Done & (b>0));

REPEAT

IF a>b THEN

a:=a-b

ELSE

b:=b-a

END;

UNTIL ((a=0) OR (b=0));

IF a=0 THEN

k:=b

ELSE

k:=a

END;

Log.Int(k);

Log.Ln

END do;

END z13_2.

В худшем случае алгоритм Евклида потребует столько же операций, что и предыдущая программа.

Ниже приведен вариант алгоритма Евклида, использующий соотношения НОД⁡ 2⁢a 2⁢b = 2⁢ НОД⁡ab , НОД⁡ 2⁢a b = НОД⁡ab при нечетном b, не включающий деления с остатком, а использующий лишь деление на 2 и проверку четности. (Число действий будет порядка log⁡n для исходных данных, не превосходящих n.)

MODULE z13_3;

IMPORT Log := StdLog, In := i21sysIn, Math;

PROCEDURE do*;

VAR a, b, k: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.Int(a);

ASSERT(In.Done & (a>0));

In.Int(b);

ASSERT(In.Done & (b>0));

k:=1;

REPEAT

IF (a MOD 2=0) & (b MOD 2=0) THEN

k:=2*k;

a:=a DIV 2;

b:=b DIV 2;

ELSIF (a MOD 2=0) & (b MOD 2=1) THEN

a:= a DIV 2;

ELSIF (a MOD 2=1) & (b MOD 2=0) THEN

b:= b DIV 2;

ELSE

IF a>b THEN

a:=a-b

ELSE

b:=b-a

END;

END;

UNTIL (a=0) OR (b =0);

IF a=0 THEN

k:=k*b

ELSE

k:=k*a

END;

Log.Int(k);

Log.Ln

END do;

END z13_3.

Пример 8.7.3. Составить программу, печатающую разложение на простые множители заданного натурального числа n.

MODULE простые1;

IMPORT Log := StdLog, In := i21sysIn, Math;

PROCEDURE do*;

VAR n, i: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.Int(n);

ASSERT(In.Done & (n>0));

Log.Int(n);

Log.Char("=");

WHILE (n#1) DO

i:=2;

WHILE n MOD i#0 DO

INC(i)

END;

Log.Int(i);

Log.Char(" ");

n:=n DIV i

END;

END do;

END простые1.

Наибольшее количество операций, равное n+C (C некоторая константа), потребуется в случае простого n. Следовательно, имеет место следующая асимптотическая оценка сложности алгоритма: f⁡(n) ∈ O⁡(n) .

Ниже приводится другой вариант алгоритма решения задачи, асимптотическая сложность которой совпадает с оценкой сложности предыдущего алгоритма.

MODULE простые2;

IMPORT Log := StdLog, In := i21sysIn, Math;

PROCEDURE do*;

VAR n, i: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.Int(n);

ASSERT(In.Done & (n>0));

Log.Int(n);

Log.Char("=");

i:=2;

WHILE (n#1) DO

IF n MOD i=0 THEN

n:=n DIV i;

Log.Int(i);

ELSE

INC(i)

END;

END;

Log.Ln

END do;

END простые2.

Составить программу решения предыдущей задачи, используя тот факт, что составное число имеет делитель, не превосходящий квадратного корня из этого числа.

MODULE простые3;

IMPORT Log := StdLog, In := i21sysIn, Math;

PROCEDURE do*;

VAR n, i: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.Int(n);

ASSERT(In.Done & (n>0));

Log.Int(n);

Log.Char("=");

i:=2;

WHILE (n#1) DO

IF n MOD i=0 THEN

n:=n DIV i;

Log.Int(i);

ELSE

IF i*i>n THEN

i:=n

ELSE

INC(i)

END;

END;

END;

END do;

END простые3.

Наибольшее количество операций потребуется для простого числа n (для всех i таких, что i2<n ). Поэтому асимптотическая оценка сложности алгоритма будет такой: f⁡(n) ∈ O⁡( n ) .

Рассмотрим многоветочный вариант цикла WHILE, введенный в программирование Дейкстрой [6, с. 296], как альтернатива цикла WHILE с вложенными циклами, призванный упростить построение и отладку сложных циклов. В версии Оберон-07 синтаксис этого цикла выглядит так:

WHILE охрана DO

последовательность операторов

{ELSIF охрана DO

последовательность операторов}

END.

В цикле выполняется та последовательность операторов, чья охрана принимает значение TRUE. Вычисление охран и выполнение последовательности операторов повторяются до тех пор, пока все охраны не примут значение FALSE.

Эффективный способ построения цикла Дейкстры заключается в том, чтобы перечислить все возможные ситуации, описывая их соответствующими охранами, и для каждой ситуации, независимо от остальных, строить последовательность алгоритмов, обеспечивающий продвижение по данным, вместе с операциями, обеспечивающими восстановление инварианта. Выход из цикла произойдет в том случае, когда все охраны примут значение FALSE.

Во многих языках программирования цикл Дейкстры конструируется. В Компонентном Паскаде достаточно использовать цикл LOOP, тело которого состоит из многоветочного условного оператора IF с оператором выхода EXIT в ветке ELSE:

LOOP

IF охрана THEN

последовательность операторов

{ELSIF охрана THEN

последовательность операторов}

ELSE

EXIT

END

END.

Пример 8.7.4 реализации алгоритма Евклида вычисления наибольшего общего делителя натуральных чисел n и m посредством цикла Дейкстры:

MODULE nod;

IMPORT Log := StdLog, In := i21sysIn;

PROCEDURE do*;

VAR n, m: INTEGER;

BEGIN

In.Open;

ASSERT( In.Done );

In.Int(n);

ASSERT( In.Done & (>>0));

In.Int(m);

ASSERT( In.Done & (m>0));

LOOP

IF n>m THEN

n:=n-m

ELSIF m>n THEN

m:=m-n

ELSE

EXIT

END;

END;

Log.String('Наибольший общий делитель равен ');

Log.Int(n);

Log.Ln

END do;

END nod.
