8.11.2. Алгоритм Кнута-Морриса-Пратта

Задача поиска строки в тексте заключается в поиске вхождения одной строки в другую строку. Самый простой и самый неэффективный алгоритм поиска вхождения одной строки в другую — линейный.

Пример реализации линейного поиска в Компонентном Паскале:

MODULE poisk_podstr;

IMPORT Log := StdLog, In := i21sysIn, Strings ;

PROCEDURE do*;

TYPE stroka=POINTER TO ARRAY OF CHAR;

VAR str, podstr: stroka;

i, j, n, m: INTEGER; bool: BOOLEAN;

BEGIN

In.Open;

ASSERT(In.Done);

NEW(str,100);

In.String(str);

ASSERT(In.Done);

n:=LEN(str$); (*определим длину строки*)

Log.String(str);

Log.Ln;

NEW(podstr,100);

In.String(podstr);

ASSERT(In.Done);

m:=LEN(podstr$); (*определим длину подстроки*)

Log.String(podstr);

i:=0;

bool:=FALSE;

REPEAT

j:=0;

WHILE (j <m) & (str[i+j]=podstr[j]) DO

INC(j)

END;

IF j=m THEN

bool:=TRUE;

ELSE

INC(i)

END;

UNTIL bool OR (i > n-m);

Log.Ln;

IF bool THEN

Log.String(podstr+' является подстрокой '+str)

ELSE

Log.String(podstr+' не является подстрокой '+str)

END;

Log.Ln

END do;

END poisk_podstr.

poisk_podstr.do "водоворот" "вор"

Пусть X = x0 … xn-1 — некоторое слово (последовательность символов), l⁡(X) — наиболее длинный префикс X, одновременно являющийся и суффиксом X, причем l⁡(X) короче X. Положим li — длина слова l⁡ ( x0 … xi ) = x0 … x li - 1 , i= 0…n-1 .

Например, l1 — длина наиболее длинного префикс-суффикса слова x0 x1 ; l2 — длина наиболее длинного префикс-суффикса слова x0 x1 x2 ; l⁡ ( aba ) = a , l⁡ ( ababa ) = aba , l⁡ ( abc ) = ∅ — пустое множество. Для X= ababc l0=0 l1=0 l2=1 l3=2 l4=0 .

Алгоритм Кнута-Морриса_Пратта для заданной строки символов X = x0 … xn-1 вычисляет длины li для всех i= 1…n-1 .

Для описания алгоритма Кнута-Морриса-Пратта понадобятся два утверждения.

Утверждение 1. Все элементы последовательности l⁡ (X) l⁡ ( l⁡ (X) ) l⁡ ( l⁡ ( l⁡ (X) ) ) … являются как суффиксами, так и префиксами слова X.

Действительно, всякий элемент последовательности l⁡ (X) l⁡ ( l⁡ (X) ) l⁡ ( l⁡ ( l⁡ (X) ) ) … по определению является как префиксом, так и суффиксом предыдущего элемента. Следовательно, всякий его элемент является как суффиксом, так и префиксом слова X.

Утверждение 2. Всякий префикс слова X, одновременно являющийся и его суффиксом, есть элемент последовательности l⁡ (X) l⁡ ( l⁡ (X) ) l⁡ ( l⁡ ( l⁡ (X) ) ) … .

Докажем утверждение 2. Пусть Y — произвольный префикс слова X, одновременно являющийся его суффиксом. Тогда Y либо совпадает с l⁡ (X) , либо является его префиксом и суффиксом, так как l⁡ (X) — самый длинный префикс X, одновременно являющийся его суффиксом. Если Y не совпадает с l⁡ (X) , то сравниваем его с l⁡ ( l⁡ (X) ) . Продолжая описанный процесс, в конечном итоге, получим элемент последовательности l⁡ (X) l⁡ ( l⁡ (X) ) l⁡ ( l⁡ ( l⁡ (X) ) ) … , совпадающий с Y.

Опишем алгоритм Кнута-Морриса-Пратта. Очевидно, что l0=0 . Пусть вычислены значения l1 … li , и требуется вычислить li+1.

Наиболее длинный префикс слова x0 … xi+1 , одновременно являющийся и его суффиксом, следует искать среди префикс-суффиксов слова x0 … xi . К слову l ⁡ x0 … xi длиной li присоединяют символ xi+1 и проверяют на предмет того, будет ли полученное слово префикс-суффиксом слова x0 … xi+1 . Если ответ отрицательный, то символ xi+1 присоединяют к слову l ⁡ ( l ⁡ x0 … xi ) и опять проверяют на предмет его совпадения с префикс-суффиксом слова x0 … xi+1 . Продолжая описанный процесс, на основании утверждений 1 и 2, получим искомый префикс-суффикс l ⁡ x0 … xi+1 .

Пример реализации алгоритма Кнута-Морриса-Пратта в Компонентном Паскале:

MODULE kmp;

IMPORT Log := StdLog, In := i21sysIn;

PROCEDURE do*;

VAR a: ARRAY 100 OF CHAR;

l: ARRAY 100 OF INTEGER;

len, i, n, dlina: INTEGER;

BEGIN

In.Open;

ASSERT(In.Done);

In.String(a);

ASSERT(In.Done);

l[0]:=0;

n:=0;

Log.Ln;

WHILE (a[n]#0X) & (n < 25) DO

Log.Char(a[n]);

INC(n)

END;

Log.Ln;

i:=0;

WHILE i#n-1 DO

len:=l[i];

WHILE (a[len]#a[i+1]) & (len > 0) DO

len:=l[len-1]

END;

IF (a[len]=a[i+1]) THEN

l[i+1]:=len+1;

ELSE

l[i+1]:=0

END;

INC(i)

END;

FOR i :=0 TO n-1 DO

Log.String('l(');

Log.Int(i);

Log.String(')=');

Log.Int(l[i]);

Log.String(', ')

END;

Log.Ln

END do;

END kmp.

Если необходимо найти вхождение слова A в слово B, то алгоритм Кнута-Морриса-Пратта можно применить к слову A#B, где # есть символ, не встречающийся ни в A, ни в B. Слово A входит в слово B, если среди чисел li,i=01…i-1, найдется число, равное длине слова A.
